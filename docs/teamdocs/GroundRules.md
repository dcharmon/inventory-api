
# Rules

## Meeting 
### Slack Every Friday

## Between meetings 
###  Slack, whenever is needed.

## Expectations when a team member needs help 
###  Communicate in Slack.

## Team members demonstrating understanding  
### Walkthrough/cold review what work they did during meetings.

## Branching and PR Strategy
### All development work is done on feature branches, never directly on main.
### Branch naming: `feature/your-feature-name`
### Every change requires a pull request with at least 1 approval before merging.
### The PR author is responsible for resolving merge conflicts on their branch before requesting review.

## What happens if there's disagreement?
### Vote for resolution

## What happens if a team member disappears?
### Split the unfinished work
